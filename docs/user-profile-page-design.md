# 유저 프로필 페이지 설계 (`/@:userId`)

> 작성일: 2026-03-21
> 대상 경로: `http://localhost:3000/@nogoso3` → (미들웨어 리라이트) → `/user/[userId]`

---

## 현재 상태 분석

### 구현된 부분

| 항목                             | 파일                                      | 상태                                 |
| -------------------------------- | ----------------------------------------- | ------------------------------------ |
| 유저 프로필 (아바타, 이름, 설명) | `user/components/user-profile.tsx`        | ✅ 완료                              |
| 팔로워/팔로잉, GitHub/홈 아이콘  | `user/components/user-profile-bottom.tsx` | ⚠️ UI만 (하드코딩 0)                 |
| 포스트 목록                      | `user/[userId]/page.tsx:50`               | ❌ 미구현 (placeholder "메인페이지") |

### 현재 렌더링 구조http://localhost:3000/@nogoso3

```
[아바타] [이름(userId) / 블로그 설명]
─────────────────────────────────
[팔로워 0 | 팔로잉 0]
[GitHub 아이콘] [Home 아이콘]
─────────────────────────────────
"메인페이지" (placeholder)
```

## 레이아웃 케이스

### Case A — 현재 구조 유지 + 포스트 목록 추가 ⭐ 추천

```
[아바타] [이름(userId) / 블로그 설명]
         [팔로워 0 | 팔로잉 0]
         [GitHub | Home 아이콘]
─────────────────────────────────
[PostCard] — 썸네일 | 제목 | 요약 | 날짜/좋아요/댓글
[PostCard]
[PostCard]
...
```

**장점**

- `PostCard` 컴포넌트(`user/components/post-card.tsx`) 바로 재사용 가능
- `page.tsx:50`의 placeholder만 교체하면 됨
- 추후 탭(Case B)으로 확장 용이

**구현 포인트**

- `postService`에 유저별 포스트 목록 API 연결 필요 (`findByUserId` 등)
- 무한 스크롤: `hooks/fetch.tsx`의 `useFetch()` 활용 (Intersection Observer + useSuspenseInfiniteQuery)

---

### Case B — 탭 기반 (포스트 / 시리즈 / 소개)

```
[아바타] [이름 / 설명 / 팔로워]
─────────────────────────────────
[포스트] [시리즈] [소개]   ← 탭
─────────────────────────────────
선택된 탭 콘텐츠
```

**URL 구조**: `/@nogoso3?tab=posts` / `?tab=series` / `?tab=about`

**장점**

- 확장성 높음 (시리즈, 소개 등 추가 콘텐츠 수용)
- velog, 브런치 등 블로그 플랫폼 표준 패턴

**단점**

- 백엔드에 시리즈/소개 API 없으면 빈 탭 발생
- `searchParams` 처리 로직 추가 필요

---

### Case C — 그리드 카드 레이아웃

```
[아바타] [이름 / 설명 / 팔로워]
─────────────────────────────────
[카드] [카드] [카드]   ← 3열 그리드
[카드] [카드] [카드]
```

**장점**

- 썸네일 중심 시각적 레이아웃 (Instagram/Brunch 스타일)
- 포스트가 많을 때 밀도 높게 표시

**단점**

- 현재 `PostCard`는 세로형 → 별도 그리드 카드 컴포넌트 신규 작성 필요
- 썸네일 없는 포스트의 빈 영역 처리 필요

---

### Case D — 사이드바 프로필 + 우측 포스트 (데스크탑 2단)

```
┌──────────────┬──────────────────────────┐
│ [아바타]      │ [PostCard]               │
│ 이름          │ [PostCard]               │
│ 설명          │ [PostCard]               │
│ 팔로워/팔로잉  │                          │
│ GitHub/Home  │                          │
└──────────────┴──────────────────────────┘
모바일: 프로필 상단 + 포스트 하단 (세로)
```

**장점**

- 데스크탑에서 공간 활용도 높음
- 프로필 정보를 항상 노출

**단점**

- 현재 `layout.tsx`에 이미 사이드바가 있어 3단 구조가 될 수 있음 → 중복 검토 필요

## 결론 및 방향

| 우선순위 | 케이스     | 이유                                      |
| -------- | ---------- | ----------------------------------------- |
| 1순위    | **Case A** | 기존 컴포넌트 재사용, 최소 구현           |
| 2순위    | **Case B** | A 완료 후 탭으로 자연스럽게 확장          |
| 보류     | Case C, D  | 신규 컴포넌트 필요 / 레이아웃 충돌 가능성 |
